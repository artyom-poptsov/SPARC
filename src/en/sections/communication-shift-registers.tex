\section{Shift registers}
\label{section:shift-registers}
\index{Electronics!Shift register}

  name=SIPO,
  description={
    Serial-In-Parallel-Out shift register.}

\newglossaryentry{PISO}{
  name=PISO,

Now that we understood bitwise operations, it's time to try them in action,
using interaction with \emph{shift registers} as an examples.  Shift registers,
also sometimes called I/O port expansions, allow us to expand the number of
digital ports on a micro-controller.  Often they are used for slow periphery
devices that have many pins -- for example, we can use shift registers to connect
seven-segment displays and arrays of buttons to a micro-controller.

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{Serial-in parallel-out (SIPO) shift register}

Serial-in parallel out (\gls{SIPO}) shift register translates a serial set of
signals (a binary code) to a parallel output.

Examples of tasks where a SIPO shift register can be used include controlling a
set of LEDs or controlling a seven-segment indicator.

One of the implementations of SIPO shift register is 74HC595 chip.  The contact
placement of the chip (in other words, its \emph{pinout}) is shown in
fig. \ref{fig:shift-register-sipo}.

\figureShiftRegisterSIPO{en}

The purpose of chip pins is described in the table
\ref{table:shift-register-sipo-pins}.

The working principle of a SIPO shift register is shown in fig.
\ref{fig:shift-register-sipo-diagram}.

\figureShiftRegisterSIPODiagram{en}

In this example we load number 37 (\mintinline{cpp}{0b00100101}) into a shift
register memory.  To do that, we sequentially set the data line (\texttt{DATA})
to bit values of the chosen number.  The shift register stores each bit into
internal memory on clock signal change from 0 (\texttt{LOW}) to 1
(\texttt{HIGH}.)  When we finish with data loading, we change the value of
(\texttt{LATCH}) line from 0 to 1, which in turn sets the voltage of Q0...Q7 to
the bit values from the shift register internal memory.

Given this description, we can develop our own procedure for data loading into a
shift register.  Let's suppose that a shift register is connected to an Arduino
as follows:

\begin{tabular}{ | m{3.0cm} | m{3.0cm} | }
  \hline
  \textbf{Digital port} & \textbf{Purpose} \\
  \hline
  2 & DS (DATA)\\
  \hline
  3 & SH\_CP (CLOCK) \\
  \hline
  4 & ST\_CP (LATCH) \\
  \hline
\end{tabular}

Firstly, we have to set one of the bits from the input data byte on the data bus
(``DS''.)  To get that bit value, we can use bitwise operations that we
discussed in the previous section.  In the current case, we can use a
combination of a bit shift and a bit mask application.

So we can write a procedure \mintinline{cpp}{shift_out} that will load a given
byte to a shift register.  An example of such procedure implementation is shown
in the listing \ref{listing:communication-sipo-shift-out}.

\begin{listing}[H]
  \begin{minted}[highlightlines={12-21}]{cpp}
    // Ports that are connected to a shift register.
    const byte DATA  = 2;
    const byte CLOCK = 3;
    const byte LATCH = 4;

    void setup() {
      pinMode(DATA, OUTPUT);
      pinMode(CLOCK, OUTPUT);
      pinMode(LATCH, OUTPUT);
    }

    // A procedure for working with a shift register.
    void shift_out(byte value) {
      for (int i = 7; i >= 0; i--) {
        digitalWrite(DATA, (value & (1 << i)) > 0);
        digitalWrite(CLOCK, HIGH);
        digitalWrite(CLOCK, LOW);
      }
      digitalWrite(LATCH, HIGH);
      digitalWrite(LATCH, LOW);
    }

    void loop() {
      shift_out(0b01010101);
    }
  \end{minted}
  \label{listing:communication-sipo-shift-out}
  \caption{A procedure for sequential loading of one byte to a shift register.}
\end{listing}

\mintinline{cpp}{shift_out} procedure accepts one byte as a parameter and
bit-by-bit shifts the byte into a register.  We are using a combination of
bitwise left shift, bitwise ``AND'' and a logical operation to move one bit into
the register.  We are performing a check if the result of bitwise operations is
greater than zero, to reduce the result to one or zero.  After setting the
\mintinline{cpp}{DATA} line to the required value, we are toggling the signal on
\mintinline{cpp}{CLOCK} which leads to the reading the current
\mintinline{cpp}{DATA} value into the register internal memory.  After the loop
finishes, we are toggling the \mintinline{cpp}{LATCH} line, outputting the the
current values from the internal register memory to the ``Q0...Q7'' pins.

Using this procedure we can control LEDs using a shift register.  In the listing
\ref{listing:communication-sipo-shift-out} we are enabling every other LED, by
specifying \mintinline{cpp}{0b01010101} bit sequence.

As another example we can implement ``Chasing lights'' effect, using bitwise
shift operations.  Aside from \mintinline{cpp}{loop} procedure, the rest of the
program (including \mintinline{cpp}{shift_out} procedure) stays the same.  In
\mintinline{cpp}{loop} procedure we can implement ``Chasing lights'' effect as
follows:

\begin{listing}[H]
  \begin{minted}{cpp}
    void loop() {
      for (int n = 0; n < 8; n++) {
        shift_out( 1 << n );
        delay(100);
      }
    }
  \end{minted}
  \label{listing:communication-sipo-chasing-lights}
  \caption{Implementation of ``Chasing lights'' effect on a shift register.}
\end{listing}

If we connect ``OE'' pin of a SIPO shift register to a digital Arduino port we
can quickly turn on and off the outputting of the values from the shift register
memory to the ``Q0...Q7'' pins, while keeping the values into the memory.  We
can use it, for example, to implement extra ``Chasing lights'' effects and even
the pulse-width modulation (PWM.)  We have already discussed PWM in
\ref{chapter:pwm} and we can use the already tried simple method of PWM
implementation (without interrupts), as is shown in
\ref{listing:communication-sipo-pwm} listing.

This example can be improved to produce more complex effects with LEDs.  Thanks
to PWM implementation using ``OE'' line, new opportunities for PWM signal
generation are opening up for us.

\begin{listing}[H]
  \begin{minted}{cpp}
    // ... Creation of "DATA", "CLOCK" and "LATCH"
    //     constants ...

    const byte OE = 5; // Output Enable

    void setup() {
      // ... Configuration of "DATA", "CLOCK" and "LATCH"
      //     ports ...

      pinMode(OE, OUTPUT);
    }

    // ... The implementation of "shift_Out" is the same ...

    // Implementation of PWM on a shift register.
    void pwm_shift_out(byte value, float dc, long t) {
      shift_out(value);
      const int T = 1000;
      int d1 = T * dc;
      int d2 = T - d1;
      int count = t / T;
      for (int n = 0; n < count; n++) {
        digitalWrite(OE, LOW);
        delayMicroseconds(d1);
        digitalWrite(OE, HIGH);
        delayMicroseconds(d2);
      }
    }

    void loop() {
      for (float dc = 0.001; dc < 1; dc += 0.001) {
        pwm_shift_out(0b01010101, dc, 1000);
      }
      for (float dc = 0.999; dc > 0; dc -= 0.001) {
        pwm_shift_out(0b01010101, dc, 1000);
      }
    }
  \end{minted}
  \label{listing:communication-sipo-pwm}
  \caption{Implementation of pulse-width modulation using a shift register.}
\end{listing}

\subsubsection{Exercises}

\begin{enumerate}
\item Create a ``Chasing lights'' effect using a shift register in such a way so
  two adjacent LEDs turn on and turn off simultaneously, using bitwise
  operations.
\item Create a ``Chasing lights'' effect using a shift register in such a way so
  LEDs are blinking starting from the center and then go to the borders, using
  bitwise operations.
\item Implement LEDs blinking effect using a shift register where each LED is
  choosing randomly.
\item Implement pulse-width modulation effect using a shift register by the
  means of fast bit switching in the shift register memory.
\end{enumerate}
