\subsection{Общие сведения}

\textit{\gls{SPI}} -- синхронная шина передачи данных, применяемая в устройствах
для связи между компонентами на коротких дистанциях.

Шина использует архитектуру ``ведущий-ведомый'', где ведущее устройство (обычно
называемое ``Master'') управляет коммуникацией с ведомым устройством (обычно
называемым ``Slave''), инициируя передачу данных.  Ведомых устройство может быть
несколько.  Некоторые устройства поддерживают смену роли ``ведущий-ведомый'' на
лету.

Протокол работает в полнодуплексном режиме, и использует четыре логических
линии:\cite{arduino:spi}

\begin{itemize}
\item \textbf{CS} (Chip Select) -- сигнал с контроллера, инициирующий
  коммуникацию с определённым устройством на шине SPI.  Активным значением линии
  является низкий уровень.  На каждое ведомое устройство требуется отдельная
  линия SS, таким образом количество этих линий звисит от количества
  периферийных устройств.
\item \textbf{SCLK} (Serial Clock) -- тактовый сигнал с контроллера.
\item \textbf{COPI} (Controller Out Peripheral In) -- Линия последовательной
  передачи данных с контроллера на периферийное устройство.
\item \textbf{CIPO} (Controller In, Peripheral Out) -- Линия последовательной
  передачи данных с контроллера на периферийное устройство.
\end{itemize}

В прошлом обычно вместо терминов ``контроллер'' (англ. \emph{controller}) и
``периферийное устройство'' (англ. \emph{periphery}) использовались термины
``мастер'' (англ. \emph{master}) и ``слейв'' (буквально ``раб'',
англ. \emph{slave}.)  В данной книге мы будем использовать современную
терминологию ``контроллер''/``периферия'' -- в таблице
\ref{table:spi-old-terminology} показано соответствия старых названий линий шины
SPI новым.

  \centering
