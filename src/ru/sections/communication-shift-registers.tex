Теперь, когда мы разобрались с битовыми операциями, пришло время их попробовать
в действии на примере работы со \emph{сдвиговыми регистрами}.  Сдвиговые
регистры, также иногда называемые расширителями портов ввода-вывода, позволяют
нам расширить количество цифровых портов на микроконтроллере.  Часто они
используются для медленной периферии с большим количеством выводов -- например,
для подключения семисегментных индикаторов и массивов кнопок к микроконтроллеру.

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{Последовательно-параллельный регистр}

Последовательно-параллельный сдвиговый регистр (\gls{SIPO}) преобразует
последовательный набор сигналов (двоичный код) в параллельный.

Примером применения подобного регистра служит управление набором светодиодов или
семисегментным индикатором.

Примером является микросхема 74HC595.  Расположение контактов (или, как говорят,
\emph{распиновка}) данной микросхемы показана на
рис. \ref{fig:shift-register-sipo}.

\figureShiftRegisterSIPO{ru}

Назначение ножек микросхемы описано в таблице
\ref{table:shift-register-sipo-pins}.

    & \textbf{Описание} \\

    \hline
  \end{tabular}

Принцип работы регистра показан на рис. \ref{fig:shift-register-sipo-diagram}.

\figureShiftRegisterSIPODiagram{ru}

В примере показана загрузка числа 37 (\mintinline{cpp}{0b00100101}) в сдвиговый
регистр.  На линии данных (\texttt{DATA}) последовательно выставляются
логические значения, соответствующие значению каждого бита загружаемого в
регистр числа.  По изменению тактового импульса (\texttt{CLOCK}) со значения 0
(\texttt{LOW}) в значение 1 (\texttt{HIGH}) происходит загрузка бита в регистр.
По завершению загрузки значение на защёлке (\texttt{LATCH}) меняется с 0 на 1,
что приводит к выставлению на ножках Q0...Q7 значений, соответствующих значениям
бит в памяти регистра.

По этому описанию мы можем разработать собственную функцию загрузки данных в
сдвиговый регистр.  Предположим, что он подключен к Arduino следующим образом:

\begin{tabular}{ | m{3.0cm} | m{3.0cm} | }
  \hline
  \textbf{Цифровой порт} & \textbf{Назначение} \\
  \hline
  2 & DS (DATA)\\
  \hline
  3 & SH\_CP (CLOCK) \\
  \hline
  4 & ST\_CP (LATCH) \\
  \hline
\end{tabular}

Нам надо выставить на шине данных (``DS'') значение одного бита из байта
информации.  Получить значения бита мы можем с использованием битовых операций,
которые мы рассмотрели в предыдущем разделе.  В данном случае, нам надо
использовать комбинацию побитового сдвига и наложения битовой маски.

Мы можем написать функцию \mintinline{cpp}{shift_out}, которая будет загружать
один байт в сдвиговый регистр.

Реализация функции \mintinline{cpp}{shift_out} показана в листинге
\ref{listing:communication-sipo-shift-out}.

\begin{listing}[H]
  \begin{minted}[highlightlines={12-21}]{cpp}
    // Порты, подключенные с сдвиговому регистру.
    const byte DATA  = 2;
    const byte CLOCK = 3;
    const byte LATCH = 4;

    void setup() {
      pinMode(DATA, OUTPUT);
      pinMode(CLOCK, OUTPUT);
      pinMode(LATCH, OUTPUT);
    }

    // Функция работы со сдвиговым регистром.
    void shift_out(byte value) {
      for (int i = 7; i >= 0; i--) {
        digitalWrite(DATA, (value & (1 << i)) > 0);
        digitalWrite(CLOCK, HIGH);
        digitalWrite(CLOCK, LOW);
      }
      digitalWrite(LATCH, HIGH);
      digitalWrite(LATCH, LOW);
    }

    void loop() {
      shift_out(0b01010101);
    }
  \end{minted}
  \label{listing:communication-sipo-shift-out}
  \caption{Функция последовательной загрузки одного байта в сдвиговый регистр.}
\end{listing}

Функция \mintinline{cpp}{shift_out} принимает один байт в качестве параметра и
побитово ``задвигает'' его в регистр.  В цикле, для занесения одного бита в
регистр используется комбинация из побитового сдвига влево, побитового ``И'', а
также логической операции.  Мы проверяем, что результат побитовых операций
больше нуля для того, чтобы свести результат к нулю или единице.  После
выставления на линии \mintinline{cpp}{DATA} нужного нам значения, мы переключаем
сигнал на линии \mintinline{cpp}{CLOCK}, что приводит к загрузке текущего
значения с линии \mintinline{cpp}{DATA} в сдвиговый регистр.  По завершению
цикла, мы переключаем значение на линии \mintinline{cpp}{LATCH}, выставляя тем
самым на ножках сдвигового регистра значения, которые находятся у него на данный
момент в памяти.

Используя данную функцию, мы можем управлять светодиодами посредством сдвигового
регистра.  В листинге \ref{listing:communication-sipo-shift-out} мы включаем
светодиоды ``через один'', через указания последовательности бит
\mintinline{cpp}{0b01010101}.

В качестве ещё одного примера мы можем создать эффект ``Бегущего огня'',
используя операцию побитового сдвига.  Кроме функции \mintinline{cpp}{loop}, вся
остальная часть программы, включая функцию \mintinline{cpp}{shift_out}, остаётся
без изменений.  В \mintinline{cpp}{loop} мы реализуем ``Бегущий огонь''
следующим образом:

\begin{listing}[H]
  \begin{minted}{cpp}
    void loop() {
      for (int n = 0; n < 8; n++) {
        shift_out( 1 << n );
        delay(100);
      }
    }
  \end{minted}
  \label{listing:communication-sipo-chasing-lights}
  \caption{Реализация эффекта ``Бегущего огня'' на сдвиговом регистре.}
\end{listing}

Если подключить ножку ``OE'' сдвигового регистра к цифровому порту Arduino, то
мы мы можем быстро включать и выключать вывод значений из памяти сдвигового
регистра на ножки ``Q0...Q7'', с сохранением текущих значений в памяти.  Это
может быть использовано, например, для реализации дополнительных эффектов
``Бегущего огня'' и даже широтно-импульсной модуляции (ШИМ.)  Мы уже разбирали
работу с ШИМ в главе \ref{chapter:pwm} и можем использовать уже испытанный нами
простой метод реализации ШИМ (без использования прерываний), как показано в
листинге \ref{listing:communication-sipo-pwm}.

Данный пример может быть доработан, чтобы получить более сложные эффекты со
светодиодами.  Благодаря реализации ШИМ через линию ``ОЕ'', перед нами
открываются новые возможности по выводу ШИМ-сигнала на большое количество
устройств.

\begin{listing}[H]
  \begin{minted}{cpp}
    // ... Создание констант "DATA", "CLOCK", "LATCH" ...

    const byte OE = 5; // Output Enable

    void setup() {
      // ... Настройка портов "DATA", "CLOCK", "LATCH" ...

      pinMode(OE, OUTPUT);
    }

    // ... Реализация функции "shift_Out" осталась прежней ...

    // Реализация эффекта ШИМ на сдвиговом регистре.
    void pwm_shift_out(byte value, float dc, long t) {
      shift_out(value);
      const int T = 1000;
      int d1 = T * dc;
      int d2 = T - d1;
      int count = t / T;
      for (int n = 0; n < count; n++) {
        digitalWrite(OE, LOW);
        delayMicroseconds(d1);
        digitalWrite(OE, HIGH);
        delayMicroseconds(d2);
      }
    }

    void loop() {
      for (float dc = 0.001; dc < 1; dc += 0.001) {
        pwm_shift_out(0b01010101, dc, 1000);
      }
      for (float dc = 0.999; dc > 0; dc -= 0.001) {
        pwm_shift_out(0b01010101, dc, 1000);
      }
    }
  \end{minted}
  \label{listing:communication-sipo-pwm}
  \caption{Реализация эффекта широтно-импульсной модуляции на сдвиговом
    регистре.}
\end{listing}

\subsection{Параллельно-последовательный регистр}

Параллельно-последовательный регистр (\gls{PISO}) преобразует параллельные
сигналы в последовательный сигнал (двоичный код.) Примером восьмибитного
параллельно-последовательного сдвигового регистра является микросхема
74HC165\cite{ti:sn74hc165}.  Распиновка данной микросхемы показана на рис.
\ref{fig:shift-register-piso}.

\figureShiftRegisterPISO{ru}

Примером применения подобного регистра служит получение данных с массива кнопок,
используя при этом меньше цифровых портов.

Назначение ножек микросхемы 74HC165 описано в таблице
\ref{table:shift-register-piso-pins}.

    \hline

    & \textbf{Альтернативные названия}
    & \textbf{Описание} \\
    \hline
    GND
    & GROUND
    & Земля, должна быть подключена к контакту ``GND'' на Arduino. \\
    \hline
    $V_{cc}$
    &
    & Напряжение питания, в диапазоне от 2В до 6В.

Реализация функции \mintinline{cpp}{shift_in} показана в листинге
\ref{listing:communication-piso-shift-in}.  Для того, чтобы считать байт из
сдвигового регистра, необходимо сначала замерять или, как говорят,
\emph{сэмплировать} (от англ. слова \emph{sample} -- ``образец''.)  Для
сэмплирования нужно выставить на линии ``LD'' значение \texttt{LOW}, затем
поменять значение на линии \texttt{CLOCK} на \texttt{HIGH}, потом на
\texttt{LOW} (что приведёт к замеру регистром значений на линиях ``A...H'' и
записи их во внутреннюю память регистра), и затем линию ``LD'' нужно подтянуть
обратно к \texttt{HIGH}.

После всех этих манипуляций мы можем считать один байт из памяти регистра,
содержащий информацию о значениях на входных ножках ``A...H'' на момент, когда
мы делали сэмплирование значений.

Чтение байта происходит побитово, каждое считанное значение добавляется в
переменную \mintinline{cpp}{result} с помощью комбинации битового сдвига и
побитового ``ИЛИ''.  Чтобы считать следующий бит из регистра, нам нужно
переключить ``CLK'' c \texttt{LOW} на \texttt{HIGH} и обратно.

Небольшие задержки в 1мкс необходимы для повышение надёжности работы с
регистром, т.к. без задержки регистр может не успевать реагировать на изменения
значений на входных линиях.

\begin{listing}[H]
  \begin{minted}[highlightlines={15-37}]{cpp}
    // Порты, подключенные с сдвиговому регистру.
    const byte DATA  = 2; // QH
    const byte CLOCK = 3; // CLK
    const byte LOAD  = 4; // SH/LD

    void setup() {
      Serial.begin(115200);
      pinMode(DATA, INPUT);
      pinMode(CLOCK, OUTPUT);
      pinMode(LOAD, OUTPUT);
      digitalWrite(LOAD, HIGH);
      digitalWrite(CLOCK, LOW);
    }

    // Функция работы со сдвиговым регистром.
    byte shift_in() {
      byte result = 0;
      // Сэмплирование значений на входах регистра.
      digitalWrite(LOAD, LOW);
      delayMicroseconds(1);
      digitalWrite(CLOCK, HIGH);
      delayMicroseconds(1);
      digitalWrite(CLOCK, LOW);
      delayMicroseconds(1);
      digitalWrite(LOAD, HIGH);
      // Чтение бит.
      for (int i = 7; i >= 0; i--) {
        byte bit = digitalRead(DATA);
        result = result | (bit << i);
        // Смещение на следующий бит.
        digitalWrite(CLOCK, HIGH);
        delayMicroseconds(1);
        digitalWrite(CLOCK, LOW);
        delayMicroseconds(1);
      }
      return result;
    }

    void loop() {
      byte b = shift_in();
      Serial.println(b, BIN);
      delay(100);
    }
  \end{minted}
  \label{listing:communication-piso-shift-in}
  \caption{Функция последовательного чтения одного байта из сдвигового
    регистра.}
\end{listing}

\subsubsection{Задачи}

\begin{enumerate}
\item Реализовать управление восемью светодиодами через
  параллельно-последовательный сдвиговый регистр и восемь кнопок.  Каждая из
  кнопок должна управлять одним из светодиодов: нажатие кнопки должно приводить
  к включению светодиода.  Повторное нажатие должно приводить к выключению.
\item Реализовать электронное пианино на основе параллельно-последовательного
  сдвигового регистра, семи кнопок и динамика.  Кнопки должны отвечать за ноты
  одной октавы (например, от С4 до B4).
\item Модифицировать предыдущее задание, добавив ещё две кнопки, для
  транспонирования (смещения) нот на одну октаву вверх или вниз.  Одна кнопка
  должна смещать все ноты на октаву вниз, а другая -- на октаву вверх.
  Установить лимит на количество шагов смещения.
\end{enumerate}

\subsection{Универсальный регистр}

Универсальный регистр может работать в обоих направлениях, в зависимости от
конфигурации.  Имеет параллельные и последовательные порты для загрузки и
выгрузки данных, а также порт для задания направления движения данных.

Примером является микросхема 74LS194\cite{ti:sn74ls194a}.

  \centering
  \small

    \hline
    \textbf{Ножка}
    & \textbf{Альтернативные названия}
    & \textbf{Описание} \\
